State Machine

Deposit States

NOT_STARTED
Initial state. No action taken.
INITIATED
Deposit request created. UPI deeplink sent or VA details shown.
IN_PROGRESS
Payment sent. Countdown timer running (~5 min). Refresh CTA shown. Home banner: "Upi Deposit - In Progress"
PENDING
"Verification Pending — can take upto 40 hours". ETA badge shown. No refresh CTA.
DELAYED
Past ETA. Delayed icon shown. May transition to trouble banner on home.
COMPLETED
Terminal state. Money credited to wallet. Green icon.
FAILED
Terminal state. Payment failed. "Verify Payment" CTA → UTR flow. Red icon. Contact Support.
PENDING state — "Verification Pending, ETA: 2 mins"
FAILED state → Trouble Banner on Home
Recovery from FAILED: User enters UTR → fetchUTRStatus API → 3 outcomes:
• should_process_deposit → auto-credit to wallet
• should_process_refund → refund to bank (user provides account + IFSC)
• cta_use_case: "contact_support" → route to Help & Support

Withdrawal States

INITIATED
Withdrawal request created after PIN + OTP verification.
PROCESSING
Bank processing the withdrawal.
DELAYED
"Withdrawal Delayed" — past expected settlement time.
COMPLETED
Money received in user's bank account.
FAILED
Withdrawal failed. May trigger refund: REFUND_INITIATED → REFUND_COMPLETED / REFUND_FAILED

ETA Calculation

getUpdatedETA({ completedAt, createdAt, etaInMins, isTerminalState }) If terminal (COMPLETED/FAILED): → Show completion timestamp If non-terminal: → eta = createdAt + etaInMins (as formatted datetime) → timeRemaining = eta - now (in seconds, for countdown) → completionTimestamp = eta datetime string Home banner uses timeRemaining for "04 min: 26 sec" display

Bank Transfer Flow — Step by Step
Bank Transfer is NEFT/RTGS/IMPS to a Virtual Account (VA) beneficiary. User adds CoinSwitch's VA as a payee in their banking app, then transfers INR. Backend auto-detects the credit.

Complete Flow: What Comes When

Step 1: User taps "Add Money"
From Home Wallet Strip ("Add" button) or Wallet Tab ("Top Up"). Navigates to Deposit Form.
Step 2: Enter Amount
Deposit Form — amount input with pill buttons (₹500, ₹1000, etc).
Per-gateway min/max validated. Gateway availability icons shown.
API: ADD_MONEY_LAYOUT(depositAmount, version:"v4")
Step 3: Select "Bank Transfer"
Gateway Selection — user picks "Bank Transfer" card (not UPI).
Card shows: availability tag, transfer methods supported, min/max limits.
API: PAYMENT_GATEWAY_LAYOUT(depositAmount, version:"v2")

Step 4: Create Virtual Account (VA)
Mutation: DEPOSIT_MUTATION_V2
POST /api/v1/custody-wallet-user/deposit
Params: { currency: "inr", amount, gateway_name: "IDBI", gateway_account_type: "BANK_ACCOUNT", generate_account: true }
Returns: additional_data containing VA beneficiary details.
Step 5: Show VA Beneficiary Details
Bank Transfer Screen — displays 1 or 2 VA accounts (primary + secondary):

Step 6: User Opens Their Bank App
User leaves CoinSwitch app and goes to their banking app (SBI YONO, Axis, ICICI iMobile, etc.).
They need to add CoinSwitch's VA as a payee/beneficiary in their bank app, then make a transfer.

If URL exists → redirect to bank. If empty → show "Add Payee" guide.
Step 7: Settlement (Auto-detected)
Backend detects the incoming NEFT/RTGS/IMPS transfer via Karnataka Bank / Indian Bank.
Credits the amount to user's CoinSwitch wallet. Transaction appears in wallet history.
Home page banner updates: "Deposit Completed".

UTR Verification Flow
UTR (Unique Transaction Reference) is a 12-digit number generated by the bank for every UPI transaction. Users need it when money is debited from their bank but not credited to the CoinSwitch wallet.

When UTR Flow Triggers

Payment Trouble Detected
Home banner: "Payment of ₹1,500 is facing trouble" OR deposit status shows FAILED/DELAYED
User taps "Verify Payment"
Routes to Payment Details screen with "Verify UTR" section
Enter 12-digit UTR Number
Input field + green submit arrow. "How to get UTR? Know More" link for education.
API: fetchUTRStatus
POST /api/v1/payment-service/deposit-details/fetch-from-utr with deposit_id + user_entered_utr
3 Possible Outcomes
should_process_deposit = true → Auto-credit to wallet
should_process_refund = true → Refund flow (user provides bank account + IFSC)
cta_use_case = "contact_support" → Route to Help & Support
