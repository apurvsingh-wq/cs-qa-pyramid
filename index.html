Gates 1-3: Compile + Lint + Static Analysis BLOCKER

The foundation. If code doesn't compile or violates conventions, nothing else runs. These run in ~30 seconds and catch typos, missing imports, type errors, and convention violations before any test executes.

AspectDetails
G1: Compileflutter analyze — zero errors, zero warnings. Catches missing imports, type mismatches, null safety violations.
G2: Lintdart format + custom rules in analysis_options.yaml. Enforces: doc comments on public APIs, Riverpod providers = *Provider, widgets = PascalCase.
G3: StaticCovered by Dart's sound null safety + flutter analyze. No implicit casts, no unused variables/imports.
WhenEvery PR + lefthook pre-push hook
Time~30 seconds

Gate 4: Unit + Widget Tests BLOCKER AI-GENERATED

The workhorse. 356 tests across 5 screens, organized into 5 layers. Generated by the /generate-tests skill which analyzes source code, extracts testable logic, and produces co-located test files with provider overrides and frozen fixtures.

5-Layer Test Architecture

LayerWhat it testsSpeedPattern
L1 Pure LogicStatic methods, math, formatting, model properties~1msexpect(Class.staticMethod(input), output)
L2 ContractfromJson factories, API shape, alternate WebSocket keys~1msModel.fromJson(loadFixture(...))
L3 ProviderState transitions, computed values, async resolution~5msProviderContainer + overrides
L4 WidgetSingle extracted widget in isolation, 4 async states~50msProviderScope(overrides: [...])
L5 PageFull page: tabs, refresh, lifecycle, composition~100msAll providers overridden, ignoreOverflowErrors()

Key Patterns Learned (20 rules)

PatternDetails
Unit > WidgetExtract private helpers to static and unit test them. Only widget-test what needs UI rendering.
Frozen fixturesLoad from test/fixtures/api_responses/ via loadFixture(). Never hardcode mock data inline.
4 async statesEvery async widget tested with: Loading, Error, Empty, Data. StreamController for loading, Stream.error for error.
Provider overrideprovider.overrideWith((ref) => fixedValue) bypasses the real dependency chain in L4/L5.
Overflow suppressionignoreOverflowErrors() must be called INSIDE testWidgets (framework resets FlutterError.onError after setUp).
Tab animationAfter tester.tap(), need pump() + pump(400ms) for TabBarView to switch content.
NavigationDon't test actual navigation (GoRouter throws without router). Verify tappable element exists instead.
Co-located testsTests live in lib/features/{name}/test/, symlinked from test/features/{name}.

Gate 5: Code Coverage Report REPORT

Coverage report generated and posted as PR comment. Doesn't auto-fail — reviewers enforce thresholds.

AspectDetails
Toolflutter test --coverage → lcov report. Per-file % and diff from base branch.
Threshold≥90% for logic files. ≥80% for UI widgets.
TraceabilityAuto-generated matrix: "X of Y Zephyr cases from this screen are covered."
Time~30 seconds (generated during Gate 4 run)

Gate 6: Golden / Visual Regression UI-SPECIFIC BLOCKER

Pixel-diff every widget against approved golden files derived from PRO web screenshots.

AspectDetails
ToolFlutter's matchesGoldenFile(). Multiple sizes: 320px (SE), 428px (Pro Max).
ThresholdFails if drift >0.5%. Update with --update-goldens.
Time~1-2 minutes

Gate 7: Accessibility Audit UI-SPECIFIC WARNING

Semantic labels, contrast ratios, tap targets ≥48px. Warnings now, blocker by v1.0.

AspectDetails
ToolFlutter guidelines: androidTapTargetGuideline + textContrastGuideline
ComplianceWCAG 2.1 AA
Time~30 seconds

Gate 8: Performance Benchmarks UI-SPECIFIC REPORT

APK size, frame rate, startup time, memory. Trading apps must hit <100ms tick-to-render.

AspectDetails
ThresholdsAPK +500KB = warning, +2MB = blocker. Startup >3s = warning. Jank >5% = blocker.
Time~5 minutes (nightly)
